# ECommerceBot — Licensing System

**Version: 1.0.0** | See also: [CUSTOMER_SETUP.md](CUSTOMER_SETUP.md) · [SECURITY.md](SECURITY.md)

## Overview

ECommerceBot uses an **offline RSA-signed license** system. Licenses are:
- Generated by the vendor using the `ECommerceBot.LicenseTool` (vendor-only)
- Validated locally without internet access
- Signed with RSA-2048 (PKCS#1, SHA-256)
- Stored in the database after activation

The private key is **never distributed** in customer packages. Only the public key is embedded in the application.

---

## License Editions

| Edition | Description |
|---|---|
| Trial | Time-limited, all features enabled |
| Standard | Production license, standard limits |
| Professional | Higher limits, priority support |
| Enterprise | Unlimited users/admins, custom bindings |

---

## License Fields

| Field | Description |
|---|---|
| `LicenseKey` | Unique identifier (16-char hex) |
| `OwnerName` | License holder's name |
| `OwnerEmail` | Contact email |
| `CustomerName` | Company name |
| `Edition` | Standard / Professional / Enterprise |
| `BotUsername` | Optional: lock to a specific bot @username |
| `AllowedDomain` | Optional: lock to a domain (e.g. example.com) |
| `ServerFingerprint` | Optional: lock to a specific server (SHA-256 hash) |
| `MaxUsers` | 0 = unlimited |
| `MaxAdmins` | 0 = unlimited |
| `IsTrial` | Whether this is a trial license |
| `ExpiresAt` | Expiry date (null = never expires) |
| `GracePeriodEndsAt` | Grace period end (set on expiry) |

---

## License Status Values

| Status | Meaning |
|---|---|
| `Valid` | License is valid |
| `Trial` | Trial license active |
| `GracePeriod` | Expired but within grace period |
| `Expired` | License has expired |
| `NotActivated` | No license found in database |
| `Invalid` | License package could not be parsed |
| `Disabled` | License deactivated |
| `SignatureInvalid` | RSA signature verification failed |
| `BotMismatch` | License bound to different bot |
| `ServerMismatch` | License bound to different server |
| `UserLimitExceeded` | Active users exceed MaxUsers |
| `AdminLimitExceeded` | Admins exceed MaxAdmins |

## Vendor: Generating Licenses

### Prerequisites

Build the License Tool:
```bash
cd tools/ECommerceBot.LicenseTool
dotnet build -c Release
```

### 1. Generate RSA Key Pair (once)

```bash
dotnet run -- keygen
```

This outputs:
- **Private key** — saved to `private_key_YYYYMMDD.pem` (keep safe, never commit)
- **Public key** — paste into customer's `appsettings.json` under `License:PublicKey`

### 2. Generate a License

```bash
dotnet run -- generate
```

The tool prompts for:
- Owner name, email, company
- Edition, bot binding, domain binding, server fingerprint
- User/admin limits
- Expiry (years)

Output: a base64-encoded license package string (e.g., `eyJsaWNlbnNlS2V5Ij...`)

Send this string to the customer. They activate it in the Telegram admin panel.

---

## Customer: Activating a License

1. Start the bot and send `/start` as an admin.
2. Press **🔐 وضعیت لایسنس** in the admin menu.
3. Press **🔑 فعال‌سازی**.
4. Paste the license package string provided by the vendor.
5. The bot validates the signature and activates the license.

## Configuration

In `appsettings.json`:

```json
{
  "License": {
    "Enabled": true,
    "PublicKey": "-----BEGIN RSA PUBLIC KEY-----\n...\n-----END RSA PUBLIC KEY-----",
    "GracePeriodHours": 72,
    "ValidateIntervalMinutes": 60,
    "RequireValidLicenseInProduction": true,
    "AllowTrial": true,
    "TrialDays": 14
  }
}
```

In `appsettings.Development.json` (override for local dev):
```json
{
  "License": {
    "Enabled": false,
    "RequireValidLicenseInProduction": false
  }
}
```

**Environment variables** (Docker):
```bash
License__Enabled=true
License__PublicKey=<public-key-pem>
```

## Server Fingerprint Binding

To bind a license to a specific server:

1. In Telegram admin panel: **🔐 وضعیت لایسنس** → **🖥 اثر انگشت سرور**
2. Copy the fingerprint hash.
3. Provide it to the vendor when requesting a server-bound license.
4. The vendor sets `ServerFingerprint` when generating the license.

---

## Grace Period

When a license expires:
- `GracePeriodEndsAt` is set to `ExpiresAt + GracePeriodHours`
- During grace period: license is still `IsValid = true` with status `GracePeriod`
- After grace period: status becomes `Expired`, webhook is blocked in production

---

## Production Protection

When `RequireValidLicenseInProduction: true`:
- `LicenseMiddleware` returns **503** for all non-health endpoints if license is invalid
- `/health`, `/health/live`, `/health/ready` always pass through

In Development:
- License validation warnings are logged but never block requests

---

## Security Notes

- The private key is **never** shipped in customer packages
- The public key only verifies — it cannot generate valid licenses
- License packages are base64-encoded JSON with RSA signature
- Signature covers all license fields (key, owner, limits, dates, bindings)
- Tampering with any field invalidates the signature
